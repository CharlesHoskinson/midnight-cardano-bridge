# CIP-0165: Canonical Cardano ledger-state root (bridge inclusion anchor)

CIP-0165 defines the **Standard Canonical Ledger State (SCLS)**: a stable,
versioned, verifiable file format for the Cardano ledger state. It is a
segmented binary container using **deterministic CBOR**, per-chunk commitments,
and a manifest, so that any two nodes at the same chain point can produce a
snapshot that hashes identically — a property Cardano's implementation-detail
serialisations do not currently guarantee. The format governs only the
**export, interchange, and verification** of state; it does not constrain a
node's internal storage.

For a trustless **Cardano → Midnight** bridge, this is exactly the missing
piece: a canonical, node-agnostic commitment to *the whole ledger state* that a
ZK proof can attest UTxO membership/nonmembership against. See the
[recursive bridge design](/bridges/midnight-cardano-recursive-bridge.md).

## The canonical state root

The `MANIFEST` record carries a `root_hash` — a **global Merkle root** — plus a
`slot_no` that pins the root to a specific blockchain slot (chain point).
Verification is a **two-level Merkle structure**:

1. **Per-namespace tree.** State is partitioned into namespaces (`utxo`, `gov`,
   `accounts`, `dreps`, `stake_pools`, …). For each namespace a Merkle tree is
   built over all **live** entry digests in **canonical key order** (tombstones
   excluded, last-writer-wins). Each entry leaf is
   `H(0x01 ‖ ns_str ‖ key ‖ value)` under **Blake2b-224**, with `0x00`/`0x01`
   domain separators on internal vs. leaf nodes.
2. **Global tree.** The global root is built over the per-namespace roots (not
   over entries directly), taken in lexicographic namespace order.

So `root_hash` is a single 28-byte digest committing, transitively, to every
live `(key, value)` entry in every namespace at a given slot.

## Why it is the bridge's Cardano-side anchor

- **Membership.** A Cardano UTxO lives in the `utxo` namespace as a
  `(key, value)` entry. A Merkle path from that entry's leaf up through the
  namespace root and then the global root proves the UTxO is included in the
  state committed by `root_hash`. This is precisely the statement a
  Cardano → Midnight inclusion proof attests.
- **Nonmembership.** Because entries within a namespace are in canonical
  (lexicographic) key order with tombstone/live-set semantics, an absent key can
  be shown by an ordered-neighbour (range) argument against the same namespace
  tree — the basis for proving a UTxO is *unspent/absent*.
- **Trustlessness.** The root is reproducible and independently signable: each
  node can verify the snapshot against its own local state and sign it,
  enabling full use of [Mithril BLS certificates](/cardano/mithril-bls-certificates.md)
  over the same `root_hash`. That certificate is what gives the Midnight side a
  succinctly verifiable, stake-weighted attestation of the anchor.

## Caveat for verifiers

SCLS files are deliberately **not byte-identical** across tools — only the stored
domain data and its hashes are canonical (this lets Mithril and other tools
append their own metadata/signature records without breaking validation). A
verifier therefore recomputes `root_hash` from the canonical data rather than
comparing raw bytes.
